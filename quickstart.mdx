title: "Quickstart: Run Your First Smart Parking Session in Java"
description: "Clone the repository, compile with Java 23, and perform a complete check-in, check-out, and availability query in under five minutes."
---

This guide walks you through getting the Smart Parking LLD system running locally and executing your first parking session end to end. You will instantiate a `ParkingManager`, check in a vehicle, collect the fee on check-out, and query real-time spot availability — all using the exact API the system exposes.

## Prerequisites

You need **Java 23** installed and available on your `PATH`. No build tool or external dependencies are required; the project compiles with `javac` directly.

<Info>
  Verify your Java version before continuing. The project uses Java records and requires Java 23 or later.

  ```bash
  java --version
  ```
</Info>

## Steps

<Steps>
  <Step title="Clone the repository">
    Clone the project from GitHub and navigate into the source directory.

    ```bash
    git clone https://github.com/bitanb1999/smartParkingLLD.git
    cd smartParkingLLD
    ```
  </Step>

  <Step title="Compile the source">
    Compile all Java source files from the project root. The entry point is `Main.java` in the root directory; role-based classes live under the `roles/` package.

    ```bash
    javac -d out $(find . -name "*.java")
    ```

    The compiled classes are written to the `out/` directory.
  </Step>

  <Step title="Instantiate a ParkingManager">
    `ParkingManager` is the single point of contact for all parking operations. The constructor takes the number of floors and the number of spots per floor. This example creates a three-floor lot with ten spots per floor.

    ```java
    import roles.*;

    ParkingManager parkingManager = new ParkingManager(3, 10);
    ```

    Internally, this initializes the spot allocator, the session database, and a fixed thread pool of ten threads used to run async operations.
  </Step>

  <Step title="Check in a vehicle">
    Call `checkIn` with a license plate string and a `VehicleType`. The method returns a `CompletableFuture<Vehicle>`. When the future completes, the returned `Vehicle` record carries the assigned spot.

    ```java
    parkingManager.checkIn("ABC123", VehicleType.CAR)
        .thenAccept(vehicle ->
            System.out.println("Assigned spot: " + vehicle.assignedSpot().spotId()))
        .exceptionally(throwable -> {
            System.err.println("Error: " + throwable.getMessage());
            return null;
        });
    ```

    <Warning>
      If no spot is available for the requested vehicle type, the future completes exceptionally with an `IllegalStateException`. Always attach an `.exceptionally()` handler so the calling thread is notified rather than silently swallowing the failure.
    </Warning>
  </Step>

  <Step title="Check out and collect the fee">
    Call `checkOut` with the same license plate used at check-in. The method returns a `CompletableFuture<ParkingTransaction>`. The transaction record contains the computed fee based on the actual parking duration and the vehicle type's hourly rate.

    ```java
    parkingManager.checkOut("ABC123")
        .thenAccept(transaction ->
            System.out.println("Fee: $" + transaction.fee()))
        .exceptionally(throwable -> {
            System.err.println("Error: " + throwable.getMessage());
            return null;
        });
    ```

    Hourly rates by vehicle type:

    | Vehicle type | Rate    |
    | ------------ | ------- |
    | `MOTORCYCLE` | $2/hr   |
    | `CAR`        | $5/hr   |
    | `BUS`        | $10/hr  |

    Checking out a license plate that was never checked in completes the future exceptionally.
  </Step>

  <Step title="Query available spots">
    `getAvailableSpots` is a synchronous call that returns the current count of unoccupied spots compatible with a given vehicle type. You can call it at any time without affecting system state.

    ```java
    int available = parkingManager.getAvailableSpots(VehicleType.CAR);
    System.out.println("Available car spots: " + available);
    ```

    <Tip>
      Use `getAvailableSpots` before calling `checkIn` if you want to surface a user-friendly "lot full" message rather than catching the `IllegalStateException` from the future.
    </Tip>
  </Step>

  <Step title="Run the program">
    Execute the compiled entry point with the `out/` directory on the classpath.

    ```bash
    java -cp out Main
    ```

    You should see output similar to:

    ```text
    Assigned spot: 2
    Available car spots: 8
    Fee: $5.00
    ```

    Exact spot IDs and fees depend on parking duration and floor layout.
  </Step>
</Steps>

## Complete example

The following is the full `Main.java` entry point, combining all the operations above in sequence:

```java
import roles.*;

class Main {
    public static void main(String[] args) {
        ParkingManager parkingManager = new ParkingManager(3, 10);

        // Check in a CAR with license plate "ABC123"
        parkingManager.checkIn("ABC123", VehicleType.CAR)
            .thenAccept(vehicle ->
                System.out.println("Assigned spot: " + vehicle.assignedSpot().spotId()))
            .exceptionally(throwable -> {
                System.err.println("Error: " + throwable.getMessage());
                return null;
            });

        // Check out the same vehicle and print the fee
        parkingManager.checkOut("ABC123")
            .thenAccept(transaction ->
                System.out.println("Fee: $" + transaction.fee()))
            .exceptionally(throwable -> {
                System.err.println("Error: " + throwable.getMessage());
                return null;
            });

        // Query how many car spots remain
        System.out.println("Available car spots: " +
            parkingManager.getAvailableSpots(VehicleType.CAR));
    }
}
```
