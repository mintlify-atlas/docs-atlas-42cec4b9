Smart Parking LLD is a complete low-level design (LLD) reference implementation for a backend parking lot management system. Written in Java 23, it demonstrates how to architect a concurrent, multi-floor parking system with automatic vehicle-type-based spot allocation, real-time availability tracking, and time-based fee calculation.

## How it works

Smart Parking LLD organizes a parking lot into floors and spots, each typed for a specific vehicle category. When a vehicle arrives, the system finds the best available spot on the lowest possible floor, records the entry time, and issues the vehicle a spot assignment. On exit, it computes a duration-based fee and frees the spot for the next vehicle.

<Steps>
  <Step title="Instantiate a ParkingManager">
    Create a manager for your lot by specifying the number of floors and spots per floor.
    ```java
    ParkingManager manager = new ParkingManager(3, 10);
    ```
  </Step>
  <Step title="Check in a vehicle">
    Call `checkIn` with a license plate and vehicle type. The system allocates the optimal spot asynchronously.
    ```java
    manager.checkIn("ABC123", VehicleType.CAR)
        .thenAccept(vehicle ->
            System.out.println("Spot: " + vehicle.assignedSpot().spotId()));
    ```
  </Step>
  <Step title="Check out and collect the fee">
    Call `checkOut` to release the spot and receive a `ParkingTransaction` with the computed fee.
    ```java
    manager.checkOut("ABC123")
        .thenAccept(tx -> System.out.println("Fee: $" + tx.fee()));
    ```
  </Step>
  <Step title="Query real-time availability">
    Check how many spots remain for a given vehicle type at any time.
    ```java
    int available = manager.getAvailableSpots(VehicleType.CAR);
    ```
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Automatic spot allocation" icon="circle-check">
    Spots are assigned by vehicle size and type, always preferring the lowest floor and lowest spot ID.
  </Card>
  <Card title="Concurrent operations" icon="bolt">
    `ReadWriteLock` and `ConcurrentHashMap` make every operation thread-safe under high load.
  </Card>
  <Card title="Async API" icon="arrows-spin">
    `checkIn` and `checkOut` return `CompletableFuture` so callers are never blocked.
  </Card>
  <Card title="Immutable data model" icon="lock">
    All entities (`Vehicle`, `ParkingSpot`, `ParkingTransaction`) are Java records — safe to share across threads.
  </Card>
</CardGroup>
